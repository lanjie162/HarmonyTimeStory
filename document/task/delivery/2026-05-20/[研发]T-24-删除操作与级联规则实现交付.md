## 2. 改动清单

### 2.1 接口层 — `AppDataStore.ets`

- 新增 `deletePerson(id: number): boolean` 签名
- 新增 `deleteStory(id: number): boolean` 签名

### 2.2 数据层 — `RepositoryImpl.ets`

- 实现 `deletePerson(id: number): boolean`
  - 事务级操作：收集 photo_ref_id → 删除 `photo_owner_link` → 递减 `photo_ref.reference_count` → 删除 `person_story` → 删除 `person`
  - 级联覆盖：Person → person_story（人物-故事关联）、photo_owner_link（照片挂载）
- 实现 `deleteStory(id: number): boolean`
  - 事务级操作：收集 photo_ref_id → 删除 `photo_owner_link` → 递减 `photo_ref.reference_count` → 删除 `person_story` → 删除 `story`
  - 级联覆盖：Story → person_story（故事-人物关联）、photo_owner_link（照片挂载）

### 2.3 内存存储 — `LocalMemoryStore.ets`

- 实现 `deletePerson`：同步级联删除 person → personStories → ownerLinks
- 实现 `deleteStory`：同步级联删除 story → personStories → ownerLinks

### 2.4 服务层 — `Services.ets` / `PersonFeature.ets` / `StoryFeature.ets`

- `IStoryService` 新增 `deleteStory(id: number): boolean`
- `PersonFeature` 添加 `deletePerson` 透传实现
- `StoryFeature` 添加 `deleteStory` 透传实现

## 3. 设计决策

| 决策 | 说明 |
|------|------|
| 事务级操作 | RepositoryImpl 使用 `beginTransaction/commit/rollBack` 保证原子性 |
| photo_ref 引用计数递减 | 使用 `MAX(0, reference_count - 1)` 防止负值 |
| 孤立 photo_ref 清理 | 在计数归零时 `SET is_referenced = 0`，保留记录（无硬删除 photo_ref）|
| 确认对话框 | 列表页和详情页均使用 `AlertDialog.show` 二次确认，防误操作 |
| 删除后导航 | 详情页删除成功后自动 `router.back()` 返回列表页 |
