## 2. 产品目标

1. **人物**与**故事**是两个 **可独立使用** 的一级能力：允许仅使用人物、仅使用故事，或两者同时使用。
2. 人物与故事之间支持 **多对多** 关联，表达「谁与哪个故事相关」，**不依赖** 是否已有照片。
3. **照片**以本机 **媒体引用** 形式管理，**不复制用户原图、不接华为云、不向自有服务端上传原图**；单张照片的归属支持 **仅人物、仅故事、或同时关联人物与故事**（通过独立关联表实现）。
4. 提供 **端侧** 智能圈选（**时间、地点元数据、人脸**），结果以 **候选 + 用户勾选确认** 方式写入（讨论稿表述；**定稿 V4.3** 已改为 **§8.4** 三形态：故事定向 / 人物定向 / 后台人脸建议，**勿**再以本节 **IM-07** 为据）。

## 3. 设计原则

| 原则 | 说明 |
|------|------|
| 存储不动 | 应用不批量落盘用户原图；持久化以引用与业务元数据为主。 |
| 模块独立 | 无人物也可建故事、加图；无故事也可维护人物与人物侧照片。 |
| 关联可选 | 人物—故事、人物—照片、故事—照片 均可为 0。 |
| 智能不越权 | 识别仅产生候选；**不得**未经确认批量写入。 |
| 人脸本地 | 人脸相关计算在端侧；单独开关与说明；特征若本地缓存须可清理并在隐私说明中描述。 |

---

## 4. 术语

| 术语 | 定义 |
|------|------|
| 人物 | 一条人物档案，可多存。 |
| 故事 | 一条故事单元，可多存。 |
| 人物—故事关联 | 多对多：一故事多人、一人物多故事；仅存关系，不代替照片挂载。 |
| 照片引用（PhotoRef） | 对本机单一媒体资源的一条应用内记录（系统标识、可选拍摄时间缓存等）；**同一系统资源在应用内对应一条 PhotoRef（去重）**，通过边表挂到故事/人物。 |
| 智能导入 | 按时间 / 地点 / 人脸（端侧）筛选候选，用户确认后加入引用关系。 |

---

## 5. 范围

### 5.1 本 MVP 包含

- 人物：列表、新建、编辑、删除；人物详情（含已关联故事、已关联照片）。
- 故事：列表、新建、编辑、删除；故事详情（元信息、有序照片、已关联人物）。
- 人物 ↔ 故事：多对多建立与解除；防重复。
- 照片：手动选图；**StoryPhoto**、**PersonPhoto** 独立维护；同一 PhotoRef 可同时挂在多故事、多人物上（每侧各自排序）。
- 智能导入：时间、地点（有则筛、无则降级）、人脸（开关、端侧）；进度、取消、失败重试；老机型降级。
- 隐私与说明页；系统删图后的失效态。

### 5.2 本 MVP 不包含

华为云及其他云相册、NAS、多人账号协作/邀请/权限、跨设备同步、服务端 AI、自动成片/PDF、疫苗与医学模板、向第三方上传原图用于识别。

## 6. 数据模型（需求层必须遵守）

### 6.1 实体

- **Person**：人物主数据（显示名必填；类型如宝宝/成人必选或枚举以定稿为准；生日可选等）。
- **Story**：故事主数据（标题必填；描述、时间范围、封面策略由实现文档锁定）。
- **PhotoRef**：本机媒体一条对应一条（按系统 URI/资源 ID 去重）；不存原图文件副本。

### 6.2 关系表

| 关系 | 含义 | 关键字段 |
|------|------|----------|
| **PersonStory** | 人物—故事多对多 | `personId`, `storyId`；可选 `role`、`sortOrder`；**唯一** `(personId, storyId)`。 |
| **StoryPhoto** | 故事—照片多对多 | `storyId`, `photoRefId`, `sortOrder`（故事内顺序）。**同一故事内** 同一 `photoRefId` 不重复。 |
| **PersonPhoto** | 人物—照片多对多 | `personId`, `photoRefId`, `sortOrder`（人物侧顺序）。**同一人物内** 同一 `photoRefId` 不重复。 |

### 6.3 删除与级联（验收依据）

- 删除 **人物**：删除该人物所有 `PersonStory`、`PersonPhoto`；**不删除** Story、`StoryPhoto`、PhotoRef（若 PhotoRef 仍被某故事引用则保留 PhotoRef 仅删人物边）。
- 删除 **故事**：删除该故事所有 `StoryPhoto`、该故事在 `PersonStory` 中的行；**不删除** Person、`PersonPhoto`、仍被其他故事或人物引用的 PhotoRef。
- 从故事 **移除** 一张照片：仅删除对应 `StoryPhoto`；若该 PhotoRef 不再被任何 Story/Person 引用，可删除 PhotoRef 或标记孤立（实现二选一并文档化）。
- 从人物 **移除** 一张照片：仅删除对应 `PersonPhoto`；对称处理孤立 PhotoRef。
- 删除 PhotoRef（若实现显式删除）：须同时清理所有 `StoryPhoto`、`PersonPhoto`。

### 6.4 与「仅人物 / 仅故事」的一致性

- 允许存在 **无任何 StoryPhoto、也无任何 PersonStory** 的 PersonPhoto（纯人物相册）。
- 允许存在 **无任何 PersonPhoto、也无 PersonStory** 的 StoryPhoto（纯故事相册）。
- **PersonStory** 可与照片挂载 **完全独立**（例如先标记「谁参与团建」再慢慢加图）。

## 7. 信息架构与导航

- 主导航包含 **人物**、**故事**（文案以实现为准）。
- 不强制「先建人物」或「先建故事」；首次启动可用轻引导说明两种入口（可选）。

## 8. 功能需求

### 8.1 人物模块（P0）

| ID | 需求 |
|----|------|
| PE-01 | 人物列表；新建、编辑、删除（删除规则见 §6.3）。 |
| PE-02 | 人物字段：显示名必填；类型、生日等以定稿为准。 |
| PE-03 | 人物详情：展示档案；**已关联故事**列表（来自 PersonStory）；**人物照片**列表（来自 PersonPhoto，有序）。 |
| PE-04 | 人物侧 **添加照片**：系统选图 → 创建或复用 PhotoRef → 写入 `PersonPhoto`（可多张、有序）。 |
| PE-05 | 人物侧 **移除照片**：仅删 `PersonPhoto`。 |
| PE-06 | 人物侧维护 **PersonStory**：关联/解除与故事的关联（可与故事侧入口二选一为主，但能力须覆盖）。 |

### 8.2 故事模块（P0）

| ID | 需求 |
|----|------|
| ST-01 | 故事列表；新建、编辑、删除（级联见 §6.3）。 |
| ST-02 | 故事字段：标题必填；描述、时间范围、封面规则实现锁定。 |
| ST-03 | 故事详情：元信息；**故事内有序照片**（StoryPhoto）；**已关联人物**（PersonStory）。 |
| ST-04 | 故事侧 **添加照片**：系统选图 → PhotoRef → `StoryPhoto`。 |
| ST-05 | 故事侧 **移除照片**：仅删 `StoryPhoto`。 |
| ST-06 | 故事侧维护 **PersonStory**：添加/移除人物；同一故事对同一人仅一条关联。 |
| ST-07 | 单故事 **StoryPhoto** 数量 **上限**（配置化），超出提示。 |

### 8.3 照片与双挂（P0）

| ID | 需求 |
|----|------|
| PH-01 | 同一系统资源对应 **一条** PhotoRef；重复添加同一资源到同一故事或同一人物时 **不重复建边**（可提示已存在）。 |
| PH-02 | 支持 **仅** 挂故事、**仅** 挂人物、**同时** 挂两者：通过分别写入 `StoryPhoto` 与 `PersonPhoto` 实现；UI 可提供「加入故事时可选同时加入人物」快捷项（**若排期不足可列为 P1**，但数据模型须 MVP 就绪）。 |
| PH-03 | 在故事/人物详情中，对同一 PhotoRef 的展示为 **同一底层引用** 的一致预览；一侧移除不自动移除另一侧（除非产品另定「联动删除」开关，MVP 默认 **不联动**）。 |

### 8.4 智能导入（P0）

**（历史）** 下表 **IM-01～IM-09** 为 **V3 讨论稿条号**，与定稿 **[§8.4.4 IM-01～IM-10](./时光故事-MVP需求说明-v4.md)** 不对照逐条等价。

| ID | 需求 |
|----|------|
| IM-01 | 在 **指定故事** 上下文中进入「智能添加」；默认写入 **`StoryPhoto`**。 |
| IM-02 | **时间**：至少自定义起止日期与单日（或等价集合，交互锁定）。 |
| IM-03 | **地点**：基于可用位置信息过滤；无位置数据归入 **未知** 或仅时间维度，不得崩溃。 |
| IM-04 | **人脸**：端侧；首次独立开关与说明；参考图或人脸簇产生候选；**仅勾选确认写入**。 |
| IM-05 | 支持时间、地点、人脸的 **组合筛选**（默认组合逻辑在技术/交互说明中写死）。 |
| IM-06 | 候选网格：全选/反选/单张剔除；确认后批量创建/复用 PhotoRef 并写入 `StoryPhoto`；写入顺序规则明确。 |
| IM-07 | **可选增强（P0 或 P1 由排期定）**：确认写入 `StoryPhoto` 时，可勾选 **同时** 写入指定人物的 `PersonPhoto`（不勾选则仅故事）。若 MVP 不做 UI，须保留接口或二期无迁移痛苦。 |
| IM-08 | 关闭人脸开关：停止新人脸索引任务；已存在引用保留。 |
| IM-09 | 长任务：进度、取消、重试；降级策略见 §9。 |

### 8.5 索引与任务（P0）

| ID | 需求 |
|----|------|
| IX-01 | 媒体/人脸索引类任务不长期阻塞 UI；可取消。 |
| IX-02 | 低端机降级（如弱化或关闭人脸、降低扫描范围）。 |

### 8.6 异常与系统相册（P0）

| ID | 需求 |
|----|------|
| ER-01 | 系统侧资源删除或不可访问：凡引用该 PhotoRef 的展示均为 **失效态**，可单条移除边或删除引用；不崩溃。 |
| ER-02 | 权限拒绝：明确提示，不死循环弹窗。 |

### 8.7 隐私与说明（P0）

| ID | 需求 |
|----|------|
| PR-01 | 静态页：不存原图副本、不上传、本地数据范围；人脸能力与开关。 |
| PR-02 | 若缓存人脸特征/索引：提供清除或与卸载说明一致。 |

### 8.8 里程碑（P1，可选）

人物下「日期 + 标题 + 备注」非医疗记录；与故事、照片无强制绑定。纳入则单独列验收用例。

## 9. 非功能需求

- **平台**：最低 HarmonyOS 版本与工程一致并文档化。
- **性能**：大相册扫描、长列表滚动、单故事照片上限可控。
- **稳定性**：选图、解码、任务取消、URI 失效路径可测。
- **语言**：MVP 可仅中文。

---

## 10. 验收检查清单（摘要）

- [ ] 仅人物：多人物、人物加图、无故事，可用。
- [ ] 仅故事：多故事、故事加图、无人物，可用。
- [ ] 人物—故事多对多：增删、防重复、删人物/删故事级联符合 §6.3。
- [ ] 同一 PhotoRef 同时存在于 `StoryPhoto` 与 `PersonPhoto`；单侧移除另一侧行为符合 PH-03。
- [ ] 手动选图去重；智能导入时间/地点/人脸及至少一条组合用例。
- [ ] （讨论稿 **IM-07**；定稿 **V4.3** 已由 **§8.4 形态二** 覆盖人物定向导入，无「顺带挂人物」单条）
- [ ] 系统删图失效态；杀进程数据一致；无云、无上传原图。
- [ ] 人脸开关与说明。

## 12. 风险与依赖

鸿蒙媒体库、PhotoPicker、端侧人脸能力以 **当前 SDK + 真机** 为准；技术设计须附 **API 与降级矩阵**。人脸准确率依赖人工勾选；地点缺失率高，需文案与 UI 预期管理。

## 13. 版本说明（相对讨论基线）

本讨论稿整合：**人物与故事独立、人物—故事多对多、PhotoRef + StoryPhoto + PersonPhoto（照片可同时挂人物与故事）、端侧智能导入（时间/地点/人脸）、不接云与不做范围外协作能力**。正文版本号 V3.0 表示相对团队内部迭代稿的语义版本；文件名为 **第一版 MVP 需求讨论稿** 供评审归档。
