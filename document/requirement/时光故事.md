二、产品定位
时光故事（TimeStory）
是一款以「故事线+人物档案」为双核心，基于鸿蒙生态打造的私人时光沉淀平台。
产品坚持：
不备份、不上传、不复制用户原图
照片保留在用户手机本地、华为云相册、NAS 私有存储
App 只做索引管理、故事聚合、人物档案、多人共享
以端侧AI实现隐私优先、高速、免费的智能整理能力
一句话定位：
让每一张照片，都有自己的故事；让每个人，都拥有完整的人生数字档案。
三、核心理念
存储不动：照片永远留在用户自有存储，App 不占空间、不做副本。
双维驱动：
故事维度：以事件为中心（旅行、聚餐、团建、生日）
人物维度：以人为中心（宝宝、自己、父母、亲友、长辈）
隐私优先：端侧AI本地运算，原图不出设备，共享最小授权。
多人共建：去中心化共享，各存各图、共编故事、全家可用。
一App双场景：亲子成长 + 通用时光，不拆分、不割裂。
四、市场痛点
原生相册只有时间地点，没有故事与记忆，照片随时间失去意义。
传统相册/亲子App强制上传原图，隐私风险高、占用空间大、不支持NAS。
家庭/聚会照片分散在多人手机，无法合并成统一故事线，共享体验差。
只有宝宝成长记录，成人无人生档案、纪念日、里程碑沉淀。
云端AI耗流量、慢、不安全，老机型体验差。
工具割裂：育儿一个App、相册一个App、共享一个App，用户成本高。
五、核心业务价值
故事化整理：把零散照片变为可回忆、可编辑、可传播的故事。
人物化沉淀：为每个人建立终身档案，包含纪念日、里程碑、参与故事。
多人协同：家庭/朋友/团队共建故事线，原图分散、故事统一。
极致隐私：不上传原图、端侧AI、最小授权、可随时撤销权限。
全存储兼容：无缝接入本地、华为云、华为家庭存储、第三方NAS。
六、目标用户
宝妈/宝爸：0-6岁儿童成长记录、家庭共享、隐私敏感。
生活记录青年：25-35岁，爱旅行、重仪式感、希望整理人生。
家庭用户：多代同堂，需要共享出游、聚会、节日回忆。
活动组织者：公司团建、同学聚会、班级活动影像归集。
NAS/私有云用户：重视数据安全，不愿原图上传第三方平台。
七、核心业务场景
1. 宝宝成长档案（亲子场景）
多宝宝独立档案
成长曲线、疫苗记录、发育里程碑
亲子故事自动归集，家庭多人共享
2. 个人时光故事（通用场景）
旅行、聚会、纪念日、生活随拍故事化整理
个人人生档案：生日、毕业、工作、重要节点
3. 家庭多人共建故事（核心壁垒场景）
一次出游，多人拍照，分散存储
共同贡献照片，合并为统一故事时间线
仅共享授权照片，不泄露个人私密相册
4. 团队/聚会集体故事（社交场景）
公司团建、年会、同学聚会
快速归集全员照片，形成集体记忆沉淀
八、核心业务能力
1. 全生态存储兼容
接入本地相册、华为云、华为家庭存储、NAS
仅引用URI索引，不复制、不迁移原图
2. 故事线管理
手动创建故事 + AI智能推荐照片
按时间/地点/场景自动聚合
故事文案、标签、标题生成
3. 通用人物档案
支持宝宝、成人、长辈、亲友
生日、纪念日、人生里程碑
自动关联该人物参与的所有故事
4. 多人去中心化共享
邀请制加入故事
成员可贡献自己相册照片
后台统一合并时间线
权限可撤销、可管理、可隔离
5. 端侧AI智能能力
本地场景识别：聚餐、旅行、户外、室内、生日等
照片智能推荐、分类、标签
老机型自动降级，保证全机型可用
6. 隐私与数据安全
本地加密存储业务数据
仅备份元数据，不备份照片
支持华为云、NAS、文件导出备份
九、技术路线（业务视角）
鸿蒙原生：基于HarmonyOS 4.0+ 开发
PhotoPicker：系统级安全选图，最小授权
端侧AI：鸿蒙Vision Kit，NPU加速、本地运算
分布式数据：多设备同步、近场同步
轻量化后台：仅存储故事、人物、索引、关系
去中心化共享：照片仍在用户设备，按需授权访问
十、商业模式
1. 基础功能永久免费
单故事创建、编辑、管理
基础端侧AI能力
单人物档案（基础版）
基础家庭共享
2. 增值付费（会员/终身买断）
多宝宝/多人物无限档案
高级云端AI、自动故事生成
无限共享故事、高级人脸聚类
NAS自动备份、成长报告导出
多设备同步、应用加密锁
十一、产品优势与壁垒
模式壁垒：不存原图、只做索引，市面独家。
双维壁垒：故事+人物，覆盖全生命周期。
共享壁垒：去中心化多人共建，无竞品实现。
生态壁垒：鸿蒙原生、深度适配华为云/NAS。
体验壁垒：隐私强、速度快、零冗余、全机型兼容。
十二、版本迭代规划
MVP（最小可用版本）
故事创建、照片索引、基础编辑
通过系统相册（PhotoPicker）引用本地与华为云相册中用户可见的媒体，不复制原图、不向自有服务端上传原图
双核：故事线 + 通用人物档案（含宝宝等类型）
端侧 **相册导入与智能关联**（定稿 **§8.4**：故事定向 / 人物定向 / 后台人脸建议；时间、地点、人脸；候选与建议均需确认；人脸簇与参考图双模式）
不做多人账号协作、邀请与权限（见 V1.1）
崩溃与错误栈上报（不上传原图、可匿名；隐私说明覆盖）
最低支持 **HarmonyOS API version 12**（详见定稿 §9.4）
