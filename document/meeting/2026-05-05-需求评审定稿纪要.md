## 2. 本轮已关闭决策（共 12 项）

| # | 主题 | 结论（摘要） |
|---|------|----------------|
| 1 | 存储 / 云相册 | 系统 PhotoPicker 引用相册可见资源（含云相册经系统呈现）；App 不调云 SDK 直读原图、不上传原图到自有服务端。 |
| 2 | MVP 范围 | 双核 + 智能导入（时间/地点/人脸）全做；多人协作不在 MVP。 |
| 3 | sortOrder | 拍摄时间降序为主；无拍摄时间则添加时间降序；MVP 无拖动重排。 |
| 4 | 最低系统版本 | 以调研表结论回填 §9.4（会后产品已裁定 **最低 API version 12**，定稿 **V4.2**、工程 `compatibleSdkVersion` **5.0.0(12)**；机型与云策略细项仍待测试阶段闭环）。 |
| 5 | PhotoRef 孤儿 | 标记孤立 + 启动/闲时 GC。 |
| 6 | 同一 URI 多挂 | 允许多故事、多人物。 |
| 7 | IM-07 | 同时挂人物 **UI 为 P1**；MVP 预留接口。 |
| 8 | PE-02 | 人物字段集定稿（含性别、头像可选等）。 |
| 9 | ST-02 / ST-07 | 故事字段集定稿；单故事 **1000** 张上限、超出阻断。 |
| 10 | IM-04 / IM-05 | 人脸簇 + 参考图双模式（默认簇）；筛选默认 **AND**。 |
| 11 | IM-06 | 写入顺序与 §6.2 一致。 |
| 12 | 列表排序 | 故事/人物列表 **最近更新降序**。 |
