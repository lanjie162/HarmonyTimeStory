## 2. 决策结论（已确认）

### 2.1 导航与全局策略

1. 首次轻引导纳入 MVP，采用一页式说明，可跳过。
2. 隐私页增加「崩溃诊断信息收集」板块。
3. 长任务采用 TaskPool 后台执行 + 通知回流进度页。
4. 形态三入口采用人物页顶栏徽章（有未读显示数量）。
5. 底栏导航采用 ArkUI `Tabs` + `barPosition(BarPosition.End)`。

### 2.2 人物侧交互

1. 头像采用独立字段 `avatarLocalPath`（不进入人物照片网格）。
2. 人物详情「已关联故事」按 `Story.startDate` 降序。
3. 人物侧入口文案由「管理」改为「关联故事」，采用 BottomSheet。
4. 入口权重：故事侧「关联人物」为主入口；人物侧保留入口但弱化。
5. 手动添加照片支持多选。

### 2.3 导入流程（形态一 / 形态二）

1. 选图后先进入「导入准备进度页」，执行 EXIF/元数据导入，再进入条件页。
2. 首次勾选人脸条件时弹出说明与开关（不在进入条件页前打断）。
3. 参考图模式：支持「选择参考照片」+ 预览 + 重选/清除；形态二额外支持「使用当前人物头像」快捷选项。
4. 取消扫描后返回条件页并保留参数，取消中展示「正在取消...」不可重复点击。
5. 地点筛选采用下拉选择器（逆地理聚合）；无数据给出说明与「未知地点」兜底。
6. 取消后已入库中间数据对用户不可见；同图再次导入时需重校验元数据，不做完全复用。

### 2.4 人脸建议（形态三）

1. 忽略后标记隐藏（不展示、不销毁），MVP 不提供回收站入口。
2. 关闭人脸开关时提供双选项：  
   - 仅关闭后续分析  
   - 关闭并清空现有建议（默认推荐）
3. 同图多人建议：主列表仅展示 top1（最高置信度），支持展开「其他候选」。
4. 阈值策略：`T_show=0.82`，`T_auto_pin=0.92`（高可信置顶但不自动写入）。
5. 批量接受采用批量事务写入；部分失败返回明细并支持重试。

### 2.5 降级与分档

1. 低端机降级保留导入主流程，降级顺序：限制扫描上限 -> 降级人脸模式 -> 关闭人脸条件。
2. 扫描上限采用机型分档：`low=100`，`mid=200`，`high=500`。
