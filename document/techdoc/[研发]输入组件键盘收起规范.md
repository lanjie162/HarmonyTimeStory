# 输入组件键盘收起规范

## 背景

HarmonyOS 上 `TextInput` / `TextArea` 在控件失焦或路由切换时**不保证自动收起 IME**（软键盘）。表现为用户完成输入、按 Done / 保存后键盘仍浮在屏幕下半，遮挡列表与导航控件。这会：

- 降低真实用户体验（需手动点按收键盘区域）
- 阻塞 Hypium 自动化测试（键盘遮挡目标控件导致点击失败）

项目初期已在 `common/ime/SoftKeyboard.ets` 中提供了 `dismissSoftKeyboard()` 工具函数。但过往编码中必须**在每个**涉及输入框的页面中手动导入、并在每个提交/navigation 入口手动调用，极易遗漏（V2.2 验收时发现了 6 处遗漏）。

---

## 解决方案：KeyboardAwareInput Builder

**位置**：`entry/src/main/ets/common/ui/KeyboardAwareInput.ets`

导出两个 `@Builder` 函数和一个包装逻辑：

```typescript
import { dismissSoftKeyboard } from '../ime/SoftKeyboard';

@Builder
export function TextInputWithKeyboard(params: TextInputParams) { ... }

@Builder
export function TextAreaWithKeyboard(params: TextAreaParams) { ... }
```

每个 Builder **自动完成**以下三件事：
1. 绑定 `onChange` 到状态变量
2. 设置 `enterKeyType(EnterKeyType.Done)`
3. 在 `onSubmit` 中先执行业务回调（如有），再无条件调用 `dismissSoftKeyboard()`

---

## 参数说明

### TextInputParams

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `placeholder` | `ResourceStr` | 是 | - | 占位文案 |
| `value` | `string` | 是 | - | 绑定的状态变量 |
| `onChange` | `(v: string) => void` | 是 | - | 变化回调，通常赋值为 `this.x = v` |
| `onSubmit` | `() => void` | 否 | `undefined` | 按 Done 的业务回调。不传时仅收键盘 |
| `height` | `Length` | 否 | `48` | 高度 |
| `layoutWeight` | `number` | 否 | `undefined` | 弹性权重。传此值时 width 自动留空 |

### TextAreaParams

参数同 TextInputParams，仅 `height` 默认值为 `80`。

## 常见场景示例

### 场景 A：多字段编辑页（不传 onSubmit）

用户填写多个字段后点击独立「保存」按钮提交。按 Done 应**仅收起键盘**。

```typescript
Column() {
  TextInputWithKeyboard({
    placeholder: '显示名',
    value: this.draftName,
    onChange: (v: string): void => { this.draftName = v; }
  })
  TextAreaWithKeyboard({
    placeholder: '备注',
    value: this.draftRemark,
    onChange: (v: string): void => { this.draftRemark = v; }
  })
  Button('保存').onClick(() => { this.saveAll(); })
}
```

### 场景 B：单字段快捷创建（传 onSubmit）

按 Done 直接执行创建。

```typescript
TextInputWithKeyboard({
  placeholder: '标题（必填）',
  value: this.newTitle,
  onChange: (v: string): void => { this.newTitle = v; },
  onSubmit: (): void => { this.createStory(); }
})
```

### 场景 C：内联输入（带 layoutWeight）

与按钮同行，使用弹性布局。

```typescript
Row({ space: 8 }) {
  TextInputWithKeyboard({
    placeholder: '输入标签',
    value: this.draftTagInput,
    onChange: (v: string): void => { this.draftTagInput = v; },
    onSubmit: (): void => { this.addTag(); },
    layoutWeight: 1,
    height: 40
  })
  Button('添加').onClick(() => { this.addTag(); })
}
```

## 违规代码示例

### 违规 1：直接使用原生 TextInput

```typescript
// 错误：不自动收键盘，需要开发者手动调用 dismissSoftKeyboard()
TextInput({ placeholder: '名称', text: this.name })
  .onChange((v: string): void => { this.name = v; })
```

修正：

```typescript
TextInputWithKeyboard({
  placeholder: '名称',
  value: this.name,
  onChange: (v: string): void => { this.name = v; }
})
```

### 违规 2：在 onSubmit 中手动调用 dismissSoftKeyboard

```typescript
// 错误：Builder 已自动兜底，手动调用是冗余的
TextInputWithKeyboard({
  placeholder: '名称',
  value: this.name,
  onChange: (v: string): void => { this.name = v; },
  onSubmit: (): void => {
    dismissSoftKeyboard(); // 多余！Builder 已做
    this.doSave();
  }
})
```

修正：

```typescript
TextInputWithKeyboard({
  placeholder: '名称',
  value: this.name,
  onChange: (v: string): void => { this.name = v; },
  onSubmit: (): void => { this.doSave(); }
})
```

## 自动化测试注意事项

Hypium / UI 自动化中，输入文本后按 Done 的步骤之后应**断言键盘已收起**：

```
// 示例断言思路
// 1. 输入文本
// 2. 按 Done
// 3. 验证底部保存按钮可见/可点击（而非被键盘遮挡）
// 4. 或验证键盘相关的 UI 树节点消失
```
