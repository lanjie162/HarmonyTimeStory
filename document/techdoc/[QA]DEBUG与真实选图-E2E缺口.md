**角色**：`qa` / `pm` / `dev` 联合维护。

## 1. 当前 Tier2 依赖的 DEBUG 形态

Hypium 主链用例（人物 / 故事 / 导入 / B2）在以下场景依赖 **BuildProfile.DEBUG** 下可见的控件，例如：

- **「【DEBUG】跳过选图进入条件（Hypium E2E）」**：绕过系统 **PhotoViewPicker** 真实选图，进入导入条件 / 扫描 / 候选步。  
- **「【DEBUG】注入相册等价 URI」 / 「再注相同 URI」**：模拟相册回调，覆盖加图与 **PH-01** 重复 URI 逻辑，**非**用户从相册点选路径。  
- **B2 负例等**：依赖 DEBUG 按钮触发 `cancelRequest` 等分支。

上述在对应 `*.test.ets` 中已有简要注释；**不等价**于上架包中的真实相册授权、多选、取消选图等体验。

---

## 2. 缺口（相对真实用户路径）

| 缺口 | 风险说明 | 建议补测方向 |
|------|----------|--------------|
| 未走 **PhotoViewPicker** | 权限拒绝、系统返回空、大图/慢机时序差异未覆盖 | 少量 **非 DEBUG** Tier2，或官方 UITest 支持的相册模拟；或与 `ued` 约定可接受的手动验收矩阵。 |
| DEBUG 仅 **debug 包** | release 渠道无该入口，但业务分支若在 DEBUG 与正式间分叉须防漂移 | 编译期开关 / Flavor 收敛（规范 §9 长期目标）；关键分支在 Tier1 可测逻辑须下沉。 |
