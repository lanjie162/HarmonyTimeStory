## 用户目标与关键场景

### 核心用户目标

1. **浏览故事**：看到故事封面、时间地点、标签和关联人物，感知全貌
2. **查看照片**：浏览该故事下的所有照片（时间降序），照片在此处是"一个事件的完整记录"
3. **关联人物**：看到故事关联了哪些人，点击跳到对应人物详情
4. **补充照片**：手动添加或批量导入更多照片到此故事
5. **编辑故事**：修改标题、描述、时间范围、地点、标签、封面

### 关键差异（与 PersonDetailPage 对比）

| 维度 | PersonDetailPage | StoryDetailPage |
|------|-----------------|-----------------|
| 照片性质 | 此人全部照片（跨多个事件） | **一个事件的照片集合** |
| 故事元素 | 混排故事封面（多个故事） | 本页就是故事本身，无其他故事穿插 |
| 关联关系 | 关联故事（多个） | 关联人物（多个） |
| 信息源 | 所有照片 + 关联故事 | 本故事 + 其照片 + 关联人物 |

### 场景与优先级

| 场景 | 频率 | 优先级 | 对应设计 |
|------|------|--------|---------|
| 浏览故事照片，回忆事件 | 高频 | P0 | 照片网格 + 封面横幅 |
| 点击关联人物头像进入人物详情 | 中频 | P0 | 人物头像行 |
| 添加少量照片 | 中频 | P1 | `[+ 添加照片]` |
| 批量导入照片 | 低频 | P1 | `[导入相册…]` → ImportPage |
| 编辑字段（标题/时间/地点/标签/封面） | 低频 | P1 | 右上角 `[编辑]` → 编辑态 |
| 删除故事 | 极低频 | P1 | 编辑态内操作 |

## 核心交互规则

### 阅览态 · 完整页面布局（ASCII 线框图）

```
┌─────────────────────────────────────────┐
│ ← 国庆游                       [编辑 ⋮] │  顶栏
├─────────────────────────────────────────┤
│ ┌───────────────────────────────────┐   │
│ │           封面横幅图               │   │  封面图，16:9 或 2:1 比例
│ │       (objectFit: Cover)          │   │  无封面时显示灰底 + 🖼️ 图标
│ └───────────────────────────────────┘   │
├─────────────────────────────────────────┤
│  2025-10-01 ~ 2025-10-07  ·  北京       │  元信息行（时间 · 地点）
├─────────────────────────────────────────┤
│  #旅行  #国庆  #家庭                    │  标签 chips（横向，溢出 +n）
├─────────────────────────────────────────┤
│  👤小明  👤小华  👤爷爷                 │  关联人物头像行（圆形 40vp，点击跳转）
├─────────────────────────────────────────┤
│ 24 张照片          [+ 添加] [导入相册…] │  照片区标题 + 操作按钮
├─────────────────────────────────────────┤
│ ┌────┬────┬────┐                        │
│ │  P │  P │  P │                        │  3 列照片网格
│ ├────┼────┼────┤                        │  （纯照片，无月份头，无故事封装）
│ │  P │  P │  P │                        │
│ ├────┼────┼────┤                        │
│ │  P │  P │  P │                        │
│ └────┴────┴────┘                        │
│              …（继续滚动）                 │
└─────────────────────────────────────────┘
```

### 封面横幅

| 项目 | 规格 |
|------|------|
| 比例 | 16:9（宽度撑满，高度自适应） |
| 无封面兜底 | 纯色灰底（`ts_surface_container_high`）+ 中间 🖼️ 图标 + "暂无封面" 文字 |
| 图片来源 | `story.coverUri`（系统图库 URI），确保 Image 组件可渲染 |
| URI 失效 | 同 V2.3 ER-01 处理方案，灰阶展示 + 图标 `已失效` |
| objectFit | `ImageFit.Cover`（裁剪填满，居中） |

### 元信息行

```
2025-10-01 ~ 2025-10-07  ·  北京
```

- 时间格式：`YYYY-MM-DD`，起止同日期时只显示一个：`2025-10-01`
- 地点：当 `story.location` 有值时才显示前缀 ` · {}`
- 两者皆空时：隐藏整行
- 字号：`ts_font_support` + `ts_text_tertiary`

### 标签 chips

```
#旅行  #国庆  #家庭  +2
```

- 使用 `Flex` 容器 + `FlexWrap.Wrap`（可换行）
- 最多显示前 3 个标签 + 溢出计数 `+n`
- 若标签总量 ≤ 3，全部显示，无溢出计数
- 若标签总量 = 0，隐藏整行
- 无标签 0 个：标签行不显示
- 每个 chip：`#标签名`，底 `ts_brand_primary_container`，字 `ts_brand_primary`，圆角 14vp

### 关联人物头像行

```
👤小明   👤小华   👤爷爷
```

- 横向排列，圆形头像 40vp
- 无头像时：`Circle` 填充 + 首字（如 `明`）缩写
- 每项由 `Image(40vp 圆形)` + 下方名字 `ts_font_overline ts_text_tertiary` 组成
- 点击头像 → pushUrl → `PersonDetailPage?personId={id}`
- 无关联人物时：隐藏整行，不占空间

### 照片网格

与 PersonDetailPage 的网格行为一致：

| 属性 | 值 |
|------|-----|
| 列数 | 3 列（`columnsTemplate('1fr 1fr 1fr')`） |
| 间距 | `columnsGap(6)` `rowsGap(6)` |
| 排序 | 拍摄时间降序（V2.2-T5） |
| 点击 | pushUrl → 全屏 PhotoSwiper（数据源为故事下的照片列表） |
| 失效态 | 灰阶 + "已失效"（V2.3 ER-01） |

**关键差异**：无月份头，无故事封面穿插——这里所有照片都属于本故事。

### 编辑态

点击右上角 `[编辑]` 后切换：

**顶部栏**：
```
← 编辑故事            [ 保存 ]
```

**内容区**（替换封面横幅 + 照片网格）：
```
┌──────────────────────────────┐
│ 标题    [_______________]     │
├──────────────────────────────┤
│ 描述    [多行文字________]   │
├──────────────────────────────┤
│ 封面    [选择封面] [移除]     │  选择 → PhotoPicker，单张选择
├──────────────────────────────┤
│ 地点    [_______________]     │
├──────────────────────────────┤
│ 开始日期  2025-10-01  📅     │  📅 弹出 CalendarPicker
│ 结束日期  2025-10-07  📅     │  📅 弹出 CalendarPicker，校验不早于开始
├──────────────────────────────┤
│ 自定义标签                     │
│  [输入标签] [添加]            │
│  #旅行 x  #国庆 x  #家庭 x   │  标签 chips，带 x 删除按钮
├──────────────────────────────┤
│ [ 删除故事 ]                  │  红色文字按钮 → AlertDialog
│  删除后将同时移除该故事的所   │
│  有关联与照片挂载，不可撤销。 │
└──────────────────────────────┘
```

**保存行为**：
- 校验标题非空 → 调用 `updateStoryAll` → Toast"已保存" → 切回阅览态 → `refresh()`
- 标题空 → Toast"标题不能为空"，保留编辑态

**日期校验**：
- 结束日期选择器：若选中的日期早于开始日期，`[确定]` 时弹出 Toast"结束日期不能早于开始日期"，不关闭对话框，要求重新选择

### 关联人物（阅览态管理入口）

阅览态中关联人物以头像行展示。如需管理关联：

- 暂定：阅览态仅展示已关联人物，**不提供"管理关联"入口**（低频操作）
- 编辑态内暂不包含关联人物管理（当前代码中"关联人物"在编辑区的下方，可在编辑态中保留该区块，不对阅览/编辑划分做强制约束）

## 全状态与异常处理

### 页面状态总表

| 状态 | 触发条件 | 阅览态表现 | 用户操作出口 |
|------|---------|-----------|-------------|
| **空态** | `this.photos.length === 0` 且 `this.linkedPersonIds.length === 0` | 封面横幅 + 元信息正常展示；照片区展示空态引导 | `[+ 添加照片]` |
| **空态 · 有照片无人** | `this.photos.length > 0` 但无人关联 | 网格显示照片，关联人物行隐藏 | 可通过编辑态关联 |
| **空态 · 有人无照片** | 有关联人物但 `photos.length === 0` | 人物头像行展示，照片区展示空态引导 | 添加照片 |
| **加载中** | 数据未返回 | skeleton：封面横幅灰块（16:9）+ 元信息灰条 + 芯片灰行 + 人物头像圆圈行 + 网格灰块区 | 加载完成自动消失 |
| **加载失败** | storyId 无效，service 返回 null | Toast"故事不存在"+ 自动 `router.back()` | 自动返回 |
| **错误态** | 故事已删除后进入 | 全页展示 `⛔ 故事不存在或已被删除` + `[返回列表]` | 返回按钮 |
| **编辑态** | 点击 `[编辑]` | 内容区切换为字段面板 | `[保存]` 或 `←` |
| **封面 URI 失效** | 系统原图已删 | 封面区灰阶 + 图标 `已失效` | 编辑态重新选封面 |
| **照片 URI 失效** | 系统原图已删 | 网格中该格灰阶 + `已失效`(V2.3 ER-01) | 可进入全屏，提供[移除关联] |

### 空态引导卡片


## 可用性优化建议

### 认知负荷

| 当前问题 | 优化 |
|----------|------|
| 编辑表单和照片区混合，用户首先看到"编辑""标题""描述"等表单 | 阅览态默认展示消费型内容（封面+元信息+标签+人物+照片），编辑折叠 |
| 两个内联 DatePicker 占据约 400vp 空间 | 改为点击弹出 CalendarPicker |
| 封面 URI 以文字截断形式显示（`file://...` 截断 30 字）可读性差 | 改为封面横幅大图展示 |
| 故事的关联人物以全量列表行的方式展示（需逐一点"关联/解除关联"）| 阅览态仅显示已关联人物的头像行（消费型），编辑态管理关联 |

### 操作步数

| 当前 | 建议 |
|------|------|
| 在故事详情看到关联人物 → 在列表中定位并点击 | 直接在阅览态头像行点击 → 跳转人物详情 |
| 编辑故事需滚动到底部找到对应字段 | Top-of-page 封面 + 元信息清晰可见，编辑态字段组织整齐 |
| 故事标签以文字形式显示在列表 | 阅览态标签以 chips 展示，编辑态可增删 |

### 文案建议

| 位置 | 当前文案 | 建议文案 |
|------|---------|---------|
| 照片区标题 | `照片（故事 · PhotoOwnerLink）` | `24 张照片`（含实际数量） |
| 添加照片按钮 | `从相册选择照片（可多选）` | `+ 添加照片` |
| 导入相册按钮 | `进入导入向导壳页` | `导入相册…` |
| 照片区空态 | `暂无挂载照片` | `这个事件还没有照片` |
| 关联人物 | `关联人物` + 说明文案 | 阅览态直接以头像行展示，无需标题 |

### 触达范围

- `[+ 添加照片]` 紧挨照片区标题右侧（拇指热区）
- 关联人物头像行水平排列，每项点击区域 ≥ 48vp（含下方名字 padding）
- 标签 chips 点击区域 ≥ 44vp（确保 chip 内边距充足）

## 交付说明与验收要点

### 版本定位

本设计为 **V2.5 交互重构版** 组成部分（L 改动量）。推荐与 PersonDetailPage 在同一版本发布，保证两块详情页体验一致。

前置依赖：

| 依赖 | 来源版本 | 状态 |
|------|---------|------|
| V2.2-C1/C2 字段扩展（Story 字段完备） | V2.2 | 已完成 |
| V2.2-T5 sortOrder（照片拍摄时间降序） | V2.2 | 已完成 |
| V2.3 ER-01 失效态照片 | V2.3 | 可后补 |

### UI 锁定点

| 项 | 锁定类型 | 说明 |
|----|---------|------|
| 封面横幅比例 | 冻结 | 16:9 固定比例 |
| 关联人物展示形式 | 冻结 | 圆形头像横向排列 + 下方名字，不允许改为文字列表 |
| 照片网格列数 | 冻结 | 3 列 |
| 阅览/编辑分离 | 冻结 | 仅顶栏 `[编辑]` 切入编辑态 |
| 标签格式 | 建议 | chips 圆角 14vp，`#标签名` 格式 |

### 可降级策略

| 场景 | 降级方案 |
|------|---------|
| 封面 URI 不存在或不可读 | 显示默认灰底兜底，不崩溃 |
| 关联人物头像缺失 | 显示 `Circle` + 名字首字缩写（如 `明`） |
| 全屏 Swiper 兼容问题 | 仅保留 `[返回]` 关闭按钮，下滑手势降级 |
| 标签过多换行（> 6 个） | 自动换行，阅读态用 +n 截断，编辑态允许滚动/展开 |
