## 设计目标与调性

**结论**：视觉定位为 **私密、可信、偏暖的本地时光相册**——克制装饰、强调 **可读层级与状态可辨**；品牌气质偏 **温润纸张 / 琥珀点缀**，避免「工具型冷灰」或「社交产品高饱和」两极。

| 维度 | 目标 | 验证方式 |
|------|------|----------|
| 调性 | 安静、可托付记忆；主界面不过度营销感 | 人物 / 故事列表首屏无强干扰色块；主操作色单一 |
| 层级 | 一屏内 2～3 级文字对比清晰 | 主标题 : 副文 : 辅助信息 对比度逐级下降且仍 ≥ 4.5:1（正文） |
| 状态 | 成功 / 警告 / 错误 / 禁用 / 信息 五色系统不混淆 | 与下文章节「语义色」对照走查 |
| 主题 | 浅 / 深两套 Token 成对维护 | 系统切换深 / 浅后主要列表、导入条、对话框可读 |

**风险**：当前工程内存在 **多处硬编码 `#RRGGBB`**（如 `ImportPage` / `PersonDetailPage`），与「Token 单一来源」冲突——**缓解**：本版在 `element/color.json` 落 **语义 Token**，新代码一律 `$r('app.color.*')`；旧代码按模块渐进替换（见 §资源与交付规格）。

## 字体与排版

**结论**：中文界面以 **系统字体** 为主；用 **字号阶梯 + 字重** 建层级，避免仅靠颜色区分信息。

| Token（`app.float.*`） | 建议值 | 用途 |
|------------------------|--------|------|
| `ts_font_display` | `22fp` | 导航标题、大标题 |
| `ts_font_title` | `20fp` | 页面标题（与现有 `20` 对齐） |
| `ts_font_headline` | `18fp` | 列表主标题、步骤标题 |
| `ts_font_body` | `16fp` | 正文、表单项 |
| `ts_font_label` | `14fp` | 表单标签、按钮 |
| `ts_font_support` | `13fp` | 辅助说明 |
| `ts_font_caption` | `12fp` | 脚注、合规小字 |
| `ts_font_overline` | `11fp` | 调试 / E2E 条（后续可收口为仅 DEBUG） |

**行高**：多行说明建议 `lineHeight` ≈ 1.35～1.5 × 字号；列表行最小触达高度 **≥ 48vp**（拇指区，对齐原型底栏 Tab）。

## 布局与关键组件视觉

| 组件 | 视觉规则 |
|------|----------|
| **底栏 Tab** | 选中态：`ts_brand_primary` 或下划线 2vp；未选中：`ts_text_tertiary`；图标与文案同色 |
| **列表行** | 左图圆角 8vp；标题 `ts_font_headline` + `ts_text_primary`；副行 `ts_font_caption` + `ts_text_tertiary` |
| **徽章（形态三）** | 数字角标：底 `ts_color_error` 或 `ts_brand_primary` 二选一全局统一；字用白或 `ts_text_primary` 反白，对比度 ≥ 4.5:1 |
| **导入步骤文案** | 保持原型「①…⑥」一条线；`ts_text_tertiary`，与主标题拉开 2 级字号 |
| **照片网格** | 间距 4vp；选中框 2vp `ts_brand_primary`；失效缩略图灰阶 + `ts_text_caption` 图标 |
| **对话框** | 主按钮填充 `ts_brand_primary`；destructive 文本 `ts_color_error`，不用填充抢主按钮 |

## 深浅色与无障碍

- **对比度**：正文与背景目标 **WCAG AA**（≥ 4.5:1）；大字标题可接受 **3:1**。  
- **禁用态**：不仅降透明度，须 **同步降字重或改 `ts_text_caption`**，避免「看起来像可点」。  
- **色觉友好**：成功 / 警告 / 错误 **辅以图标或文案**，不单靠色相。  
- **大字号**：跟随系统 `fontSizeScale` 时，导入条与列表须 **允许折行**，禁止固定单行截断无 tooltip。
