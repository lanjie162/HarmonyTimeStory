## 用户目标与关键场景

### 核心用户目标

1. **浏览此人**：一眼看到此人的照片、关联故事和基本信息，无需滚动/点击就能感知内容量
2. **回忆时光**：按时间顺序自然浏览照片，中间的"故事封面"唤起对事件的完整记忆
3. **补充内容**：快速添加照片到此人物，或从相册批量导入
4. **修改变更**：修改姓名、生日、类型等字段（低频率操作）
5. **了解关联**：查看此人与哪些故事关联，并一键跳转故事详情

### 关键场景与优先级

| 场景 | 频率 | 优先级 | 对应设计 |
|------|------|--------|---------|
| 点击人物列表行，浏览该人物的照片 | 高频 | P0 | 阅览态时间线网格 |
| 从时间线中发现故事封面，点击进入故事详情 | 中频 | P0 | 故事封面穿插 + Card 底标条 |
| 添加少量照片（PhotoPicker 单次选择） | 中频 | P1 | `[添加照片]` 按钮 |
| 批量导入照片（相册向导） | 低频 | P1 | `[导入相册…]` 按钮 → ImportPage |
| 修改人物字段（生日、类型、备注等） | 低频 | P1 | 右上角 `[编辑]` → 编辑态 |
| 全屏慢慢翻看照片 | 中频 | P0 | 全屏 Swiper |
| 删除人物（不可逆） | 极低频 | P1 | 编辑态内操作 |

### 选区 · 摘要行

| 元素 | 行为 |
|------|------|
| 头像 | 圆形 64vp，点击无跳转（头像操作为编辑态功能），长按无操作 |
| 类型标签 | 纯文本展示（同 `typeLabel()` 映射），不可点击 |
| 生日文本 | 纯文本展示，不可点击（编辑在编辑态完成） |
| 照片计数 / 故事计数 | `128 张照片 · 3 个故事` 格式，纯文本，**可提取为变量**（反应最新数据） |
| `[+ 添加照片]` | 触发 PhotoPicker，单次选择后直接写入（PE-04），写入后刷新网格 |
| `[导入相册…]` | pushUrl → ImportPage（形态二，personId 已锁定），面包屑显示人物名称 |

**计数变量**：
- 照片计数：`this.photos.length`（当前已挂载的 PhotoRef 数量）
- 故事计数：`this.linkedStoryIds.length`（已关联的故事数量）

### 时间线网格

**排序逻辑**：
- 所有元素（照片 + 故事封面）按其"时间锚点"统一降序排列
- 照片时间锚点：`taken_at_cache`（拍摄时间）→ 回退至添加时间（V2.2-T5 已实现）
- 故事封面时间锚点：故事的 `timeStart`（故事开始时间）
- 故事封面仅在其时间锚点所属月份头下方出现一次，不重复穿插

**月份头粘性**：
- `2025 年 10 月` 格式文本，沿左对齐
- 使用 ArkUI 粘性分组机制（如 `List` 的 `sticky` 属性或手动检测滚动位置 + 悬浮定位）
- 滚动跨月时，旧月份头被新月份头推出顶部

**网格列数**：
- 默认 3 列（`columnsTemplate('1fr 1fr 1fr')`）
- 单行最低高度：~96vp（照片高度）
- 间距：`columnsGap(6)` `rowsGap(6)`

### 故事封面 · 视觉区分规则

封面图在网格中占用 1 个 GridItem，与照片同尺寸，但附加以下区分特征：

```
┌──────────────────┐
│                  │
│   故事封面图      │
│                  │
│   ─────────────  │  ← 半透明渐变（底→上）
│   📖 国庆游      │  ← 白色文字标签，右下角
└──────────────────┘
```

**三层视觉区分**（优先级由高到低，至少实现前两个）：

1. **标签 badge**：封面图右下角（或左下角）叠加圆角矩形标签，文字 `📖 故事名`，底 `ts_brand_primary` 色半透明（~60% opacity），字白色 11fp
2. **色调区分**：封面图叠加极淡品牌色蒙层（~8% opacity `ts_brand_primary` 覆盖层）
3. **圆角区分**：故事封面圆角 16vp，照片圆角 8vp（仅当前两条已实现时作为辅助）

**禁用状态**：当故事封面 URI 失效时（V2.3 ER-01），封面区显示灰阶占位 + 图标 `🖼️` + 标题文字兜底

### 故事底标条

- 网格中每个故事封面所在的**那一行之后**，紧跟一条"底标"
- 底标行不换行，左对齐文字 + 右箭头 `→`
- 格式：`📖 {故事名} · {n} 张  →`
- 点击效果同封面图：pushUrl → StoryDetailPage
- 若连续两行出现同一故事封面（理论上不会，按去重设计），只在前一行后出底标

### 编辑态

点击右上角 `[编辑]` 后，页面切换：

**顶部栏**：
```
← 编辑人物            [ 保存 ]
```

**内容区**（替换时间线网格）：
```
┌──────────────────────────────┐
│  ⭕ 头像占位     [更换] [移除]│
├──────────────────────────────┤
│ 显示名  [_______________]    │
├──────────────────────────────┤
│ 类型    宝宝 成人 长辈 其他   │  按钮组，选中态品牌色填充
├──────────────────────────────┤
│ 生日    2024-08-15  📅       │  📅 图标点击 → CalendarPicker 弹出
├──────────────────────────────┤
│ 性别    ○男  ○女  ○未填     │
├──────────────────────────────┤
│ 备注    [多行文字________]   │
├──────────────────────────────┤
│ [ 删除人物 ]                  │  红色文字按钮（非填充），触发 AlertDialog
│  删除后将同时移除该人物的所有 │
│  关联与照片挂载，不可撤销。   │
└──────────────────────────────┘
```

**保存行为**：
- 校验显示名非空 → 调用 `updatePerson` → Toast"已保存" → 切回阅览态 → `refresh()` 刷新数据
- 校验失败 → Toast 提示原因，保留编辑态

**取消行为**：
- 点击 `←` 返回 → 无确认对话框（无未保存的草稿丢失风险，因字段已实时存入 draft state）
- 若希望更安全：可在退出编辑态前执行 `refresh()` 重置所有 draft 为原始值

### 生日选择（CalendarPicker）

**阅览态**：生日文本仅展示，不可点击（编辑态才可修改）

**编辑态日历弹出**：
- 用户点击生日行上的 `📅` 图标 or 点击生日文本
- 弹出自定义 `CustomDialog`，内含 `CalendarPicker` 月视图
- 选中日期后对话框不自动关闭，需用户点 `[确定]` 或 `[取消]`
- 确定后更新生日文本，对话框关闭
- 不可选未来日期（最大值为今天）（若需限制，由 domain rules 决定）

**CalendarPicker 视觉对齐**：
- 选中态：圆底 `ts_brand_primary` 填色，字白色
- 今天：圆圈描边 `ts_brand_primary`
- 月份/年份切换：Dialog 内左右箭头
- 整体 Dialog 背景色 `ts_surface`

## 全状态与异常处理

### 页面状态总表

| 状态 | 触发条件 | 阅览态表现 | 用户操作出口 |
|------|---------|-----------|-------------|
| **空态** | `this.photos.length === 0` 且 `this.linkedStoryIds.length === 0` | 摘要行正常展示；网格区展示空态引导卡片 | `[+ 添加照片]` 添加第一张照片 |
| **空态 · 有故事无照片** | `this.photos.length === 0` 但 `linkedStoryIds > 0` | 摘要行正常；网格区只显示故事封面 + 底标条 | 点击故事封面跳转 |
| **加载中** | `aboutToAppear` / `onPageShow` 数据尚未返回 | 摘要行展示 skeleton placeholder（浅灰圆+浅灰文字条 2 行）；网格区展示 6 格灰块 skeleton（skeleton.shimmer 效果可选） | 等待加载完成自动消失 |
| **加载失败** | Service 返回 null（person 不存在） | 全页 Toast"人物不存在" + 自动 `router.back()` 到列表页 | 自动返回 |
| **错误 · 人物删除后进入** | 已删除的人物 ID 通过路由参数进入 | 全页展示：`⛔ 人物不存在或已被删除`；仅一个 `[返回列表]` 按钮 | 点按钮回到列表 |
| **离线** | 无网络（本机应用不需要网络，此状态不适用） | 不处理 | — |
| **编辑态** | 点击 `[编辑]` | 内容区切换为字段编辑面板 | `[保存]` 或 `←` 取消 |
| **编辑中 · 键盘展开** | 点击文本输入框 | 滚动行为需配合键盘高度调整（已有 KeyboardAwareInput 组件） | 键盘 Done / 点击空白收起 |
| **照片 URI 失效** | 系统图库图片已删除（V2.3 ER-01） | 该网格项展示灰阶 + 图标 `已失效` + 文字"图片已不可访问" | 可点击进入全屏查看无法加载的画面；全屏中提供 `[移除关联]` 按钮 |

## 可用性优化建议

### 认知负荷

| 问题 | 建议 |
|------|------|
| 当前详情页查看和编辑混在一起，用户被大量表单干扰 | 阅览/编辑分离，默认只展示消费型内容（照片+故事） |
| 照片区被深埋在第 5 个区块 | 时间线网格放在摘要行后立即展示，无需滚动即见首行 |
| 故事以文字列表形式出现在照片上方，信息量低 | 故事以封面图穿插在时间线中，视觉占比更高，触发浏览兴趣 |

### 操作步数

| 当前 | 建议 | 步数变化 |
|------|------|---------|
| 添加照片：滚动到页面底部 → 点"从相册选择照片" | 摘要行直接 `[+ 添加照片]` 按钮 | 5步 → 2步 |
| 查看故事详情：滚动到故事区 → 找到故事行 → 点击 | 直接点击时间线中的故事封面 | 4步 → 1步 |
| 修改生日：滚动到编辑区 → 滚动 DatePicker → 滚轮选日期 | 编辑态点📅 → 日历点选 → 确定 | 5步 → 3步 |

### 文案建议

| 位置 | 当前文案 | 建议文案 | 理由 |
|------|---------|---------|------|
| 添加照片按钮 | `从相册选择照片（可多选）` | `+ 添加照片` | 简短，意图明确，省去括号说明 |
| 批量导入按钮 | `进入导入向导壳页` | `导入相册…` | "壳页"是开发术语，用户看不懂；"导入相册"更生活化 |
| 故事底标 | `关联故事` + 列表文字 | `📖 国庆游 · 12 张  →` | 一眼看出故事名和照片量，有浏览欲 |
| 空态引导 | `暂无挂载照片` | `还没有照片\n添加照片或导入相册到这里吧` | 更友好，给出操作出口 |
| 照片计数摘要 | (无) | `128 张照片 · 3 个故事` | 用户进页第一个感知——"这个人有多少内容" |

### 触达范围

- `[+ 添加照片]` 置于摘要行右上区域（拇指热区）
- 编辑入口 `[编辑]` 置于顶栏右上角（标准位置）
- 故事底标条中的 `→` 右侧加 padding 保证点击区域 ≥ 48vp

## 交付说明与验收要点

### 版本定位

本设计为 **V2.5 交互重构版** 核心模块，预计 XL 改动量。前置依赖：

| 依赖 | 来源版本 | 状态 |
|------|---------|------|
| V2.2-T5 sortOrder（照片拍摄时间降序） | V2.2 | 已完成 |
| V2.2-C5 taken_at_cache（缺省时间兜底） | V2.2 | 已完成 |
| V2.2-T1 字段扩展（Person 字段完备） | V2.2 | 已完成 |
| V2.3 ER-01 失效态 | V2.3 | 可在时间线之后补充 |

### UI 锁定点

| 项 | 锁定类型 | 说明 |
|----|---------|------|
| 时间线网格列数 | 冻结 | 3 列，不允许改为 2 列（照片密度降低）或 4 列（缩略图过小） |
| 故事封面视觉区分方案 | 冻结 | 标签 badge 必做，色调蒙层推荐，圆角区分可选 |
| 阅览/编辑态切换入口 | 冻结 | 仅顶栏右 `[编辑]` 按钮，不允许双击、长按等隐式入口 |
| 编辑态内删除入口 | 冻结 | 红色文字按钮，非填充按钮，避免误触 |
| 月份头格式 | 冻结 | `YYYY 年 M 月`（如 `2025 年 10 月`），不加"份"字 |

| # | 检查项 | 判据 | 类型 |

| P-13 | 保存/取消闭环 | 编辑态保存后切回阅览态且数据更新；点击 `←` 放弃编辑恢复到前值 | 功能 |
| P-14 | 照片计数/故事计数 | 摘要行显示准确的 `{n} 张照片 · {m} 个故事` | 数据 |
