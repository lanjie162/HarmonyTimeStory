## 模块 A · Tabs 原生导航

### 现状问题

`MainTabBar.ets` 使用 `router.replaceUrl` 在 `PersonPage` / `StoryPage` 间切换，导致：
- 每次切换 Tab 页面状态丢失（滚动位置、输入框内容等归零）
- 需要中间 `ShellPage` 做路由中转，路径冗余
- 与 ArkUI 原生 `Tabs` 相比，缺少手势滑动切换 Tab 的能力

### 设计方案

**改 Tabs 组件为 ArkUI 原生 `Tabs` + `TabContent`**。

### 顶层容器（ShellPage 重构）

```
┌─────────────────────────────────────┐
│        (无独立顶栏 — Tab 内自带)     │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  TabContent: PersonPage    │    │
│  │  (直接嵌入，非 router)     │    │
│  │                             │    │
│  └─────────────────────────────┘    │
│                                     │
│  ┌─────────────────────────────┐    │
│  │  Bottom Bar                 │    │
│  │   [ 人物 ]      [ 故事 ]     │    │  ← Tabs bar
│  └─────────────────────────────┘    │
└─────────────────────────────────────┘
```

**结构示意（伪代码）**：

```
Tabs({ index: currentIndex, barPosition: BarPosition.End }) {
  TabContent() {
    PersonPage()     // 组件直接引用，保持状态
  }
  .tabBar('人物')

  TabContent() {
    StoryPage()      // 同理
  }
  .tabBar('故事')
}
```

### 行为规则

| 场景 | 行为 |
|------|------|
| 点击 Tab "人物" | index=0，平滑切换到人物列表（保持之前的滚动位置、搜索/输入状态） |
| 点击 Tab "故事" | index=1，同理 |
| 手指横向滑动 | 支持手势滑动切换 Tab（Tabs 默认行为） |
| 从详情页 `router.back()` 返回 | 返回到当前 Tab 下的列表页，Tab 索引不改变 |
| 详情页来回路由 | 保持现有 `router.pushUrl` 模式（PersonDetailPage / StoryDetailPage 仍然独立路由） |

### 详情页返回行为

| 从详情页返回 | 效果 |
|-------------|------|
| PersonDetailPage → back | 回到 PersonPage（Tab 索引=0） |
| StoryDetailPage → back | 回到 StoryPage（Tab 索引=1） |
| 两者切换 | 通过 pushUrl 跨进对方详情页后 back，回到自己的列表页 |

### 移除项

- 从 `PersonPage.ets` 移除顶栏"返回"按钮（Tab 切换本身就是导航行为）
- 从 `StoryPage.ets` 移除顶栏"返回"按钮
- 移除 `ShellPage.ets` 的中间中转逻辑

### 保留项

- `MainTabBar.ets` 移除，不再需要（由 Tabs 原生组件替代）
- `PersonPage` / `StoryPage` 的内部内容保持，仅移除"返回"按钮行

### 视觉规格

| 属性 | 值 |
|------|-----|
| 底栏背景 | `ts_surface_container_high` |
| 选中 Tab 文字 | `ts_brand_primary` + `FontWeight.Medium` |
| 未选中 Tab 文字 | `ts_text_tertiary` + `FontWeight.Regular` |
| 选中指示器 | 下划线 3vp 宽度 28vp（保持现有风格） |
| Tab 高度 | 52vp（拇指热区） |

## 模块 B · CalendarPicker 生日/日期选择器

### 适用范围

| 页面 | 字段 | 说明 |
|------|------|------|
| `PersonDetailPage` 编辑态 | 生日 | 单选日期 |
| `StoryDetailPage` 编辑态 | 开始日期 | 单选日期 |
| `StoryDetailPage` 编辑态 | 结束日期 | 单选日期，需要校验 >= 开始日期 |

### 交互流程

```
阅览态                   编辑态                      弹出后
┌──────────┐           ┌──────────┐           ┌──────────────────┐
│ 生日      │           │ 生日     │           │   选择生日        │
│ 2024-08  │  [编辑]   │ 2024-08  │  📅点击   │                   │
│ -15      │  ─────→   │ -15 📅   │  ─────→   │  ← 2024年8月  →  │
│          │           │          │           │ ┌─┬─┬─┬─┬─┬─┬─┐ │
│ (不可点)  │           │ (可点击) │           │ │ │ │ │ │1│2│3│ │
│          │           │          │           │ ├─┼─┼─┼─┼─┼─┼─┤ │
│          │           │          │           │ │4│5│6│7│8│←15│ │
│          │           │          │           │ ├─┼─┼─┼─┼─┼─┼─┤ │
│          │           │          │           │ │16│17│...│   │ │
│          │           │          │           │ └─┴─┴─┴─┴─┴─┴─┘ │
│          │           │          │           │   [取消] [确定]   │
└──────────┘           └──────────┘           └──────────────────┘
```

### 状态表

| 状态 | 阅览态 | 编辑态·未操作 | 编辑态·弹出日历中 | 编辑态·已选择新值 |
|------|--------|-------------|-----------------|-----------------|
| **未设置生日** | 显示"未设置"灰字 | 显示"未设置"灰字 + 📅 | 日历默认值：`2000-01-01` | 文本更新为新日期 |
| **已有生日** | 显示日期 `2024-08-15` | 显示日期 + 📅 | 日历默认值：已有日期 | 文本更新为新日期 |
| **选择中** | — | — | 日历高亮已选日期，用户可更改 | — |

### CalendarPicker 视觉规格

基于鸿蒙 `CalendarPicker` 组件，定制样式对齐视觉系统：

| 属性 | 值 |
|------|-----|
| 弹出容器 | `CustomDialog`（居中弹窗），背景 `ts_surface`，圆角 16vp |
| 选中日期 | 圆底 `ts_brand_primary` 填色，字白色 |
| 今日日期 | 圆圈描边 `ts_brand_primary` |
| 非本月日期 | 灰色不可选（可禁用） |
| 未来日期 | 可禁选（取决于 domain rules） |
| 月份切换 | 左右箭头 `<` `>` |
| 年份快速跳转 | 点击年份文字展开年选择器（可滚动的 1900~当前年） |
| 确定按钮 | 主填充 `ts_brand_primary`，字白色 |
| 取消按钮 | 文字按钮，`ts_text_secondary` |
| 尺寸 | Dialog 宽度 ~320vp，高度自适应（不超出屏幕上下安全区） |

### 日历弹出层边界状态

| 场景 | 行为 |
|------|------|
| 选择日期后点确定 | Dialog 关闭，Text 更新为新日期 |
| 点取消 | Dialog 关闭，Text 保持为进入编辑态时的值 |
| 点 Dialog 外部 | 同取消效果（Dialog 关闭，值不变） |
| 起止日期校验失败 | **不关闭** Dialog，底部 Toast "结束日期不能早于开始日期" |
| 生日选择未来日期 | 若 domain rules 允许，不做限制；若不允许，未来日期灰色不可点 |

## 模块 C · PhotoSwiper 全屏浏览组件

### 组件定义

```
@Builder / @Component
PhotoSwiper(params: {
  photos: PhotoRefModel[],   // 照片数据源
  initialIndex: number,      // 从第几张开始 (0-based)
  onRemove?: (photo: PhotoRefModel) => void  // 移除回调（可为空）
})
```

### 交互流程

```
┌──────────────────────────────────────┐
│  ← 2024-08-15          3/24   [⋮]   │  顶部透明栏
├──────────────────────────────────────┤
│                                      │
│                                      │
│            ┌──────────────┐          │
│            │    照片       │          │  Swiper 内容区
│            │              │          │  objectFit: Contain
│            │  (上下黑边)   │          │
│            └──────────────┘          │
│                                      │
│                                      │
├──────────────────────────────────────┤
│    [ 已失效 ] 或 照片正常加载         │  底部状态行（可选）
└──────────────────────────────────────┘
```

### 行为规则

| 手势/操作 | 行为 |
|-----------|------|
| **左右滑动** | 切换照片（Swiper 的 loop 模式：首张向左 → 末张，末张向右 → 首张） |
| **向下滑动** | 关闭 Swiper，回到照片网格（Swiper 的 `onTouch` 手势 + 动画关闭） |
| **点击顶部`[返回]`** | 关闭 Swiper（同下滑） |
| **点击`[⋮]`** | 弹出 BottomSheet 操作菜单 |
| **右滑或上滑** | 保持 Swiper 不响应额外方向（只响应左右和向下） |

### 操作菜单（BottomSheet）

```
┌──────────────────────────────────┐
│   🖼️ 照片选项                    │
├──────────────────────────────────┤
│   [ 从人物移除 ]                  │  红色文字，触发确认对话框后删除
│   （"/从故事移除"，按语境动态变化）│
└──────────────────────────────────┘
```

### 视觉规格

| 属性 | 值 |
|------|-----|
| 背景色 | 全屏黑色 `#000000` |
| 顶部栏 | 透明渐变底（底→上渐变为 10% 黑→60% 黑），24vp 高 |
| 返回文字 | 白色 16fp `←` + 日期 |
| 页码 | 白色 12fp `3/24` |
| 照片 | `objectFit: ImageFit.Contain`（保持比例，上下留黑边） |
| 失效态照片 | 居中显示灰底 🖼️ 图标 + "图片已不可访问"文字 |

### 状态表

| 状态 | 表现 |
|------|------|
| **正常加载** | 照片全屏展示，比例为 Contain |
| **加载中** | 居中 loading indicator（圆环） |
| **加载失败/URI 失效** | 灰底 + 图标 `🖼️` + 文字"图片已不可访问" |
| **单张照片** | 页码显示 `1/1`，左右滑动 loop 保持但实际不变 |
| **空数据源** | 组件不应被调用（上游保证 `photos.length > 0`） |

## 模块 D · 列表页增强

### D1 · PersonPage 列表增强

**阅览态列表行**（改造后）：

```
┌───────────────────────────────────────┐
│                                       │
│  ┌────┐  小明                          │
│  │ 头  │  宝宝 · 男                    │
│  │ 像  │  128 张 · 3 个故事            │
│  └────┘  更新于 昨天                  │
│                                       │
└───────────────────────────────────────┘
```

**行规格**：

| 元素 | 规格 |
|------|------|
| 头像 | 圆形 48vp；无头像时 `Circle` 填充 `ts_surface_container_high` |
| 显示名 | 18fp `ts_text_primary` `FontWeight.Medium` |
| 类型·性别 | 13fp `ts_text_tertiary`（同现有 typeLabel + genderLabel） |
| 计数摘要 | 12fp `ts_text_caption` `128 张 · 3 个故事` |
| 更新日期 | 12fp `ts_text_muted` `更新于 昨天` |
| 整行高度 | ≥ 72vp（含上下 padding） |

**空态行**保持不变，文案不变。

**创建入口改造**：
- 移除"新建人物"输入框卡片（行 111~139）
- 右上角顶栏 `[+ 创建]` 按钮 → 点击弹出 **BottomSheet**

**创建 BottomSheet**：
```
┌──────────────────────────────────┐
│   新建人物                        │
├──────────────────────────────────┤
│  [________________________]      │  显示名输入框（自动聚焦）
│  [ 创建 ]                       │  品牌色填充按钮
└──────────────────────────────────┘
```

- 输入框自动聚焦
- 按回车 / 点"创建" → 创建人物 → 关闭 BottomSheet → 刷新列表 → Toast "已创建"
- 点击 BottomSheet 外部 → 关闭，输入框内容丢弃

**滑动删除**：
- 列表行支持 `SwipeAction` 左滑露 `[删除]`（红色文字按钮）
- 点击删除 → 现有 `AlertDialog` 确认 → 确认后删除 → 刷新列表
