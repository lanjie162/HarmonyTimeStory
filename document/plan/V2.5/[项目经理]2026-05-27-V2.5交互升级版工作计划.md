## 2. 计划策略

### 目标与范围

**目标**：V2.5 不阻塞于 V2.4，将 MVP 全部主页面从"功能可用的工程原型"升级为"交互规范的消费级应用"。UED 已于 2026-05-23 交付全部设计。

### In 范围（7 模块）

| 模块 | 摘要 | 设计来源 |
|------|------|---------|
| A | Tabs 原生导航：ShellPage 改为 Tabs+TabContent，替换 router.replaceUrl；移除 MainTabBar.ets | 全局组件规格 §A |
| B | CalendarPicker 日历选择器：CustomDialog 封装鸿蒙 CalendarPicker，替换 PersonDetailPage（生日）和 StoryDetailPage（起止日期）共 3 处 DatePicker | 全局组件规格 §B |
| C | PhotoSwiper 全屏浏览：新建 common/ui/PhotoSwiper.ets，支持左右滑动/下滑关闭/操作菜单 | 全局组件规格 §C |
| D | 列表页增强：PersonPage + StoryPage 改为 BottomSheet 创建、SwipeAction 左滑删除、行规格重设计 | 全局组件规格 §D |
| E | PersonDetailPage 阅览/编辑态分离 + 时间线混排（照片+故事封面穿插 + 月份头粘性 + 故事底标条 + P-01~P-16） | PersonDetailPage 时间线混排设计 |
| F | StoryDetailPage 阅览/编辑态分离 + 封面横幅(16:9) + 元信息行 + 标签chips + 关联人物头像行 + S-01~S-16 | StoryDetailPage 事件相册设计 |
| G | 视觉 Design Token 替换：逐模块使用 `$r('app.color.ts_*')` 替换裸色；新代码零裸色；全仓扫尾 | 视觉系统设计 v1 |

### Out 范围（本版不做）

- ImportPage / SuggestPage / PrivacyPage / SettingsPage 不改造
- 时间线虚拟化（>500 张优化为 Future TODO）
- 导入流程交互改造

### 执行顺序

```
批次1（可插空 V2.4）     批次2（V2.5 前半）      批次3（V2.5 主包）
┌─────────────┐       ┌───────────────┐       ┌─────────────────┐
│ 模块 B (S)  │       │ 模块 A (M)    │       │ 模块 E (XL)     │
│ CalendarPicker│ →   │ Tabs 原生     │ →     │ PersonDetailPage│
└─────────────┘       │ 模块 D (M)    │       │ 模块 F (L)      │
                       │ 列表页增强    │       │ StoryDetailPage │
                       └───────────────┘       │ 模块 C (M)      │
                                                │ PhotoSwiper     │
                                                │ 模块 G (S)      │
                                                │ 视觉 Token      │
                                                └─────────────────┘
```

### 复杂度评估口径

采用 S/M/L/XL，与主计划一致。
