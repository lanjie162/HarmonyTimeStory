## 2 · 冲突陈述

- **需求 / 技术权威**：[需求 §6](../../requirement/时光故事-MVP需求说明-v4.md)、[技术方案 §4](../../techdoc/时光故事-MVP技术方案-v1.md) 要求业务实体 **持久化**（默认 RDB）。
- **V2 执行与验收口径**：主链以 `LocalMemoryStore` 为会话真相；部分 V2 归档将「**落库**」解释为「会话内数据结构落账、**非 DB 文件**」。
- **用户 Must（2026-05-14）**：杀进程 / 冷启动后，人物、故事、`PersonStory`、`PhotoOwnerLink` 等已提交数据 **可读回且一致**。

上述与用户 Must **不可同时成立**于产品验收层，须升格持久化闸门。

### 3.1 新基线（产品验收）

以 **V2.1-DoD**（[主计划 §4 · V2.1](../2026-05-06-MVP研发工作计划-定稿.md)）为准：

| 编号 | 要点 |
|------|------|

| V2.1-D3 | 写→杀进程→冷启→读 L2 证据 |

| V2.1-D6 | pm/qa 联合结论 → V3 准入输入 |

### 3.2 废止范围（仅限产品口径）

- **废止**：在 **V2 联合验收** 中，将「落库」等同于「**非持久化 / 仅内存落账**」的 **产品验收解释**。
- **不废止**：V2 主链 UI、导入步骤、B2/C9 自动化等 **工程证据价值**；历史归档正文 **不删改**（见 [追溯索引](../../evidence/V2.1/2026-05-20-T21-cr01-traceability-index.md) §V2 索引）。
