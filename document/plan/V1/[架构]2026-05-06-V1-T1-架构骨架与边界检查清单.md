## 2. 分层职责与依赖基线（冻结）

### 2.1 层级职责

- `pages/`：ArkUI 页面与路由壳层，仅承载展示和轻编排。  
- `features/`：业务编排层，对外暴露 `features/api/`。  
- `infrastructure/`：系统与数据适配层，对外暴露 `infrastructure/api/`。  
- `domain/`：纯模型与规则，无 IO，可被所有层引用。  
- `bootstrap/`：唯一装配点，负责实现实例化与依赖注入。

### 2.2 依赖方向

1. `pages` 只能依赖 `features/api` 与 `domain`。  
2. `features` 只能依赖 `infrastructure/api` 与 `domain`。  
3. `infrastructure` 可依赖 `domain` 与 `@ohos.*`。  
4. `bootstrap` 是唯一允许同时看见 `api` 与实现目录的区域。

## 3. 边界检查清单（V1-T1 必交）

> 判定规则：每条检查项结论为 `通过 / 不通过 / 有条件通过`，并附最小证据。

| 编号 | 检查项 | 检查方法 | 通过标准 | 失败处置 |
|------|--------|----------|----------|----------|
| T1-C1 | `pages` 不直连 `infrastructure` | import 扫描 + 代码评审 | `pages/**` 无 `infrastructure/**` import | 修复 import，必要时经 `features/api` 转接 |
| T1-C2 | `pages` 不依赖 `infrastructure/api` | import 扫描 | `pages/**` 无 `infrastructure/api` 引用 | 同上 |
| T1-C3 | `pages` 仅依赖 `features/api` + `domain` | import 扫描 | 无 `features/*` 实现目录依赖 | 回收至 `features/api` 暴露接口 |
| T1-C4 | `features` 不 import `@ohos.*` | import 扫描 + 评审 | `features/**` 无 `@ohos.*` | 下沉到 `infrastructure` 封装 |
| T1-C5 | `features` 不直连 `infrastructure` 实现目录 | import 扫描 | `features/**` 无 `infrastructure/(repository|media|vision|task|data)` import | 经 `infrastructure/api` 抽象 |
| T1-C6 | `features/infrastructure` 实现不引入 ArkUI 组件 | 评审 | 无 ArkUI 页面组件依赖 | UI 逻辑回收到 `pages` |
| T1-C7 | `bootstrap` 为唯一实现装配点 | 目录评审 + 代码评审 | 仅 `bootstrap/**` 出现实现装配代码 | 统一迁移回 `bootstrap` |
| T1-C8 | `domain` 保持无 IO | 评审 | 无系统 API、无存储/网络调用 | 将 IO 迁移至 `infrastructure` |

## 6. 版本边界声明（V1）

1. V1-T1 只保证“骨架正确 + 依赖边界正确 + 可检查可追溯”。  
2. 不以 V1-T1 承诺业务流程可用性与体验完整度。  
3. 对边界规则的临时例外，必须登记并在 V1 结束前清零；未清零不得进入 V2。

---

## 7. V2 输入条件（架构维度）

进入 V2 前需同时满足：

1. `T1-C1~T1-C8` 全部通过，或仅存在已批准且有关闭时限的例外。  
2. `bootstrap` 装配点唯一性成立。  
3. `features/api` 与 `infrastructure/api` 可支撑 stub/mock 替换。  
4. `domain` 共享模型命名稳定，可被 V2 主链复用。

---

## 8. 风险与升级机制

| 风险 | 触发信号 | 影响 | 处置 |
|------|----------|------|------|
| 边界规则形同虚设 | 连续发现跨层直连 | 后续改造成本陡增 | 当日修复 + 追加扫描门禁 |
| bootstrap 外泄 | 多处出现装配代码 | 可替换性下降 | 统一回收到 `bootstrap` |
| 例外长期不清理 | 例外跨版本遗留 | V2 输入污染 | 升级决策点，冻结新功能 |
