## 2. 全局治理规则

### 2.1 真机验证规则（MCP）

1. 真机前置验证统一使用 `deveco-mcp`。  
2. 含真机任务的版本，退出条件必须包含 MCP 证据（`UI tree` / `hilog` / `faultlog` / `build log` 适用项）。  
3. 若 MCP 工具链阻塞达到阈值，触发项目级决策点，由用户裁定范围调整/暂缓策略。  
4. 证据命名统一：`版本-任务-设备-时间`；目录统一：`evidence/<version>/<task>/`。

### 2.2 条件触发规则（B1）

- `C7` 用 `deveco-mcp` 验证 `R-01`。  
- 若 `R-01` 不成立，触发 `B1` 并入当前版本。  
- 并版护栏：仅允许回修与 `R-01` 直接相关的 MVP `§5/§6` 及对应技术方案段落；超范围必须新开决策点。

### 2.3 联合验收冲突规则

- 各版本由 `pm` 与 `qa` 联合验收。  
- 若结论冲突，自动升级为决策点提交用户裁决；版本状态冻结为「待裁决」。

## 4. 版本内任务计划

### V1（底座版）

| 任务编号 | 任务内容 | 复杂度 |
|----------|----------|--------|
| V1-T1 | 建立 `pages/features/infrastructure/domain/bootstrap` 分层骨架与依赖边界，并交付边界检查清单 | L |
