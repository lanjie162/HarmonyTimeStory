# Skill: task-drive（task 包 · 编排元技能）

> **Skill 元信息**
>
> - **属于**：[`manager`](../../../manager.md)（中立位编排；**非**原子层 skill）
> - **层级**：task 包 · **编排元技能**（正文**可**点名 `task-execute` / `task-accept` / `task-quality` 以对齐实质规则；**不得**替代三者的守门与写列契约）
> - **目的**：对指定的一条或多条 `T-n`，按 §当前在跑 **真实状态** **按需**串联 **执行 → 验收 → 质检**（`task-execute` → `task-accept` → `task-quality`），并在每子步 **执行前 / 执行后** 输出公示块，便于复盘；默认在可配置粒度下 **暂停等你确认** 再进入下一子步
> - **不包含**：`task-dispatch`、`task-broadcast`（需通晒时另起一轮对应 skill）
> - **入口字面**（示例）：「跑 task-drive」「`skill: task-drive`」「任务流水线」「按流水线推 T-12,T-13」
> - **专属编号**：不新增；沿用 `T-n` / `B-x`
> - **主入参**：`T-n` 列表或筛选条件（由你在开跑明示）+ 可选 **小结粒度** + 可选 **批量同环节** + 可选 **发言来源**
> - **主出参**：§当前在跑 按原子 skill 规则被合法更新（若本轮完成子步）；本 skill 外加固定公示块与小节

## 与本包其它 skill 的分工（强制）

| 能力 | `task-drive`（本 skill） | 原子 skill |
|------|---------------------------|------------|
| 守门与允许写入列 | 编排层**委派**对齐，不得放宽 | [`task-execute`](task-execute.md) / [`task-accept`](task-accept.md) / [`task-quality`](task-quality.md) **各自独占** |
| 跨 skill 字面互指 | **允许**写明「对齐 `task-execute` …」 | 正文 **互不直接引用** 对方 skill 名（仍为仓库约定） |
| 「推进一组且无流水线字面」路由 | **不**顶替 `manager.md` **按状态直走原子** | 由 `manager` 分组后直接调某一原子 |

> 总则、§任务请求契约 / §统一任务列表契约 / §来源选型 / §节奏 / §子代理正文绑定：一律以 [`manager.md`](../../../manager.md) 为准；本 skill **不重述**，仅标明触发节拍。

## §角色装载（强制 · 实为读档而非版式）

> **本节为硬门槛**：不满足则视为 **违背本 skill**；不得以「文风上写了视角」顶替。

1. **定义**：**装载** = 在本助手回合产出 **实质性角色块 / 守门 / 改表** 之前，已对对应正文完成 **等价于工具 `Read` 的全文拉取**，并在会话中打出 **`【task-drive · 装载】`** 公示块。**禁止**只写模板字句但未读文件即发言。
2. **公示格式（每笔子步必选，须在「即将执行」之后、视角块之前紧接输出）**

   ```text
   【task-drive · 装载】
   - 已读取：<仓库相对路径 `.cursor/agents/<name>.md`> → 本步责任：<角色标签>：<与 task-execute | task-accept | task-quality 哪一段对齐一句话>
   ```

3. **分环节硬性要求**
   - **execute**：**恰好一条**装载行：`owner_role` 对应 `.cursor/agents/<owner_role>.md`。
   - **accept**：**每位** `acceptor_role` **各一轮**装载行：**仅读取** `.cursor/agents/<acceptor>.md`。（验收仍属专业角色口吻，不靠未读 md 臆造。）
   - **quality**：**恰好一条**装载行：`已读取：.cursor/agents/manager.md` → 本步：**中立位守约 + [`task-quality.md`](task-quality.md) 全流程（含 `【task-quality · manager 装载】`、`【manager · 质检论证】`（Q1～Q6）、状态扭转）**。**禁止**再读第二份 `.cursor/agents/<其它角色>.md` 或使用 Task 充当「第二质检员」。首次进入 quality 的回合内必须完成本条 **Read + 装载公示**。
4. **`【task-drive · 本步实况】`** 须含一项 **`装载回看：与上方【task-drive · 装载】一致`**（批量模式可简称「已装载·同计划」但必须可核对路径）。

[`manager.md`](../../../manager.md) §子代理正文绑定 其余款（缺失补位等）对本节 **叠加适用**。

## 阶段零 · 启动节奏（调度 + 计划公示）

### 步骤 1 · Skill 调度

按 `manager.md` **调度公示**，首行：

`【Skill 调度】skill: task-drive；依据：<用户原话 / 路径 / 关键词>；后续将按 .cursor/agents/manager/skills/task/task-drive.md 执行。如需更改请回复「改用 <其他 skill>」。`

### 步骤 2 · 执行前总公示（强制）

紧随其后输出 **`【task-drive · 计划】`**，须可逐项核对：

- **范围**：本回合 `T-n` 列表及处理顺序（或筛选条件）；`document/task/[任务经理]统一任务列表.md` 路径锚点
- **每条快照**：自表中读取的 **状态**、`owner_role`、`acceptor_role[]`（验收顺序预览）、本条 **将跳过 / 将执行** 的环节（由状态推导）
- **运行选项**（须在开跑与用户对齐或在此公示缺省）：
  - **小结粒度**：**默认** = accept **每位**一节 `【task-drive · 小结】`；execute / quality **每环节**一节；你可在开跑明示 **「仅环节级小结」** 则 accept 全流程结束后只汇总一次（仍须每位会签守门顺序不变）
  - **批量例外**：明示「批量 / 本节不停」→ 可于连续子步省略「请回复继续」，但 **`【task-drive · 即将执行】`/「本步实况」省略须不得省略守门结果与表更要点**
  - **发言来源**：主会话模拟 / Task 取证 / 沿用全流程（`manager.md` §来源选型）
- **预计子步序列**：对每个 `T-n` 给出有序清单，每项形如 `execute→task-execute` / `accept(k=…)→task-accept` / `quality→task-quality`

若范围或语义不清：**仅最短一问**收窄（例如列出候选 `T-n` 让用户确认）。

### 步骤 3 · 范围复核

与用户确认：**是否按上述计划开局**（默认等你一句「继续」；批量模式跳过等待但计划块仍必须先发）。

## 状态路由（按需跳过 · 须在计划中体现）

对每条 `T-n`，**每一步子步开始前重读** §当前在跑 该行（防守竞态）。

| 读到状态 | 下一可执行流水线环节 | 跳过说明 |
|-----------|---------------------|----------|
| `待执行` | execute（对齐 `task-execute`） | — |
| `已执行`、`验收中` | accept（对齐 `task-accept`）；多人时按 `acceptor_role[]` 顺序逐角色 | execute 跳过 · **`【task-drive · 跳过】`** |
| `已验收` | quality（对齐 `task-quality`，含其 **来源选型**） | execute / accept 跳过 · **`【task-drive · 跳过】`** |
| `已完成`、`已通晒` | （无） | 整条 **`【task-drive · 跳过】`**：`T-n` 无需本流水线 |

**多条 `T-n` 的顺序**：默认按你在 **计划中** 给出的列表 **纵向切段**（对每条尽量走到 quality 可走的最远点后，再根据小结粒度停顿）；你若明示 **「批量同环节」**，则先做所有条的 execute，再做所有条的 accept……（仍为每步子步守则 + 公示）。

## 每子步固定节拍（强制）

对 **计划中列出的每一子步**（含被跳过情形），顺序输出下列块；允许将 **「本步实况」首节与「小结」合并**，但 **守门结果 + 表更要点**不得省略。

### A · 【task-drive · 即将执行】（子步前）

在 **§角色装载** 的读档动作之前输出（可与装载紧接在同一回合前半段）：

- `T-n`、`环节`（execute | accept | quality）、对齐原子 skill（`task-execute` / `task-accept` / `task-quality`）及阶段范围（典型 **阶段一至二**，阻塞走阶段三）
- **责任角色**：owner / 「acceptor 第 k 位：`<role>`」/ quality：**固定 manager**（同回合须与 **§角色装载** 中 `.cursor/agents/manager.md` 行一致）
- **预期写表**：本轮**允许**改的列（仅该原子允许的列）；**明确不触碰**的列名
- 若本子步 **整体跳过**：改发一行 **`【task-drive · 跳过】`**：`T-n`、当前状态、跳过理由

### A′ · 【task-drive · 装载】（紧随 A，子步前 · 强制）

内容及顺序 **严格按上文「§角色装载」**；**本条缺省视为本回合未完成子步**。quality 环节 **禁止**跳过 **manager.md** 装载行。

### B · 角色装载与实质推进（对齐原子）

1. **`【task-drive · 装载】` 已输出且对应文件已完成读档**（见 §角色装载）。
2. 按 §来源选型：主会话 **`【某某视角】`** 或 Task 取证（须在实况中标注）；视角标题中的 **某某**须与装载行路径中的 `<name>` 一致。
3. **实质行为规范**须与下列文件 **对应阶段**一致：
   - execute → [`task-execute.md`](task-execute.md) 阶段一至三  
   - accept → [`task-accept.md`](task-accept.md) 阶段一至三（每位 acceptor 一遍）  
   - quality → [`task-quality.md`](task-quality.md) 全流程（含 **§条目 Tn · 来源**、**fixed(manager)**、`Q1～Q6`、状态扭转）

### C · 【task-drive · 本步实况】（子步后）

- **`装载回看`**：**与本子步「【task-drive · 装载】」所列路径一致**（quality 子步仅 `manager.md` 一行）。
- **对齐原子 skill**、**守门**（通过 / 未通过「缺项类目」）
- **表更**：§当前在跑 **是否写入**；**状态** `old→new`；已写入列的 **一行摘要**（勿贴全表）
- **证据**：1～3 条短路径/锚点（无则写「本轮无外链证据」）
- **发言来源**：主会话模拟 / Task（quality 子步与 **manager** 装载一致，**不**派 Task 给其它角色做质检）

### D · 【task-drive · 小结】（停顿点 · 批量可压缩提示语）

默认模板：

```text
【task-drive · 小结】
- 范围：T-<n>（可加 来源任务请求反链）
- 刚完成：环节=<execute|accept|quality>；子步=<角色|会签第k位>
- 表更：状态 <old> → <new>；列摘要 <…>
- 证据：… 
- 风险/阻塞：<B-x 或守门未通过摘要>
- 下一默认步：<一句话>
```

末行决策提示（批量模式可改为单行「本子步已连续推进；需要时可随时喊停」）：

`请回复「继续」进入下一子步，或说明调整（改验收顺序 / 改用 Task / 暂停 / 改处理其它 T-n）。`

**默认**：在你回复 **`继续`** 前 **不得**开启下一 `【task-drive · 即将执行】`（你已声明批量除外）。

## 多人验收顺序（accept）

- **顺序**：默认与 §当前在跑 表中 `acceptor_role[]` 列字面顺序一致；你可开跑明示重排。
- **每角色**：独立完成 `task-accept` 守门与会签语义（首位 / 中间 / 末位扭转规则不变）。
- **小结**：默认 **每角色**一节 `小结`（若选「仅环节级」则最后一位后再统一 `小结`，但 **`本步实况` 仍每位一条**）。

---

## 质检（quality）

进入 `task-quality` 环节时：**不压缩** [`task-quality.md`](task-quality.md) 自带的 **来源选型**停顿点（可与 `小结`交错）。**且**须 **同时遵守** 上文 **§角色装载**：quality **仅** `manager.md` 一行；[`task-quality.md`](task-quality.md) **`【task-quality · manager 装载】`** 与 **`【task-drive · 装载】`** 须在 **同一回合内**指向同一读档事实。

## 回合收束（可选）

范围含多条 `T-n` 或长会话结束前，可加：

**`【task-drive · 回合摘要】`**：按时间列出本子回合 `(T-n · 环节 · 角色 · 状态变化)`，一行一条。

## 阻塞与打回

- 任一步命中 **`B-x`**、`task-quality` DoD 上游问题、`task-accept` 冲突升级：**停止自动串联**，交由 `manager` 中立裁决；本 skill **不中转**业务结论。
- 打回到 `待执行` / `已执行`：**在实况与小结中指明下一原子 skill**，用户确认后再入环。
