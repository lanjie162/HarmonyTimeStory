# Skill: task-accept（公共技能）

> **Skill 元信息**
>
> - **属于**：公共技能（可由 `pm / sa / dev / qa / ued / manager` 按职责调用）
> - **层级**：task 包 · **验收原子**
> - **目的**：将统一任务列表 §当前在跑 中状态为 `已执行 / 验收中` 的任务推进到 `已验收`
> - **核心语义**：`acceptor_role[]` **直接提交**验收结论；本技能负责会签聚合、守门校验与状态扭转
> - **入口**：
>   - 角色自助提交：「qa 提交 T-12 验收结论」
>   - 用户点名提交：「验收 T-13，并按模板会签」
> - **专属编号**：不分配新编号；引用 `T-n` / `B-x`
> - **主入参**：单条 `T-n`（状态=`已执行` 或 `验收中`）+ 结构化验收结论
> - **主出参**：`验收结论` 列追加 + 状态扭转（`已执行 → 验收中/已验收`）

## 信息边界（强制）

> 引用 [`manager.md`](../../../manager.md) §任务请求契约 → 信息边界。本技能不向上游产物写入 `T-<数字>`。

## 角色自驱会签模型

| 角色 | 职责 |
|------|------|
| `acceptor_role[]` | 直接提交验收结论并对结论负责 |
| 调用方（任意角色） | 触发本技能并传入材料，不改变验收责任归属 |
| 本技能 | 聚合会签、守门校验、写 `验收结论` 列、扭转状态、冲突升级 |

## 多人会签语义

- 单人验收：`已执行 + 通过 -> 已验收`
- 多人验收：
  - 首位提交后：`已执行 -> 验收中`
  - 中间位提交：保持 `验收中`
  - 末位提交且全为通过：`验收中 -> 已验收`
  - 任一结论为不通过：保持当前状态并升级阻塞

## 验收守门（全过才可进入已验收）

1. **结论明确**：`通过 / 不通过 / 条件通过` 三选一
2. **依据可寻址**：每条结论对应执行证据定位
3. **不通过可定位**：必须指向 DoD 条目与缺口
4. **条件可执行**：条件通过必须有条件清单与时限
5. **范围对齐**：验收覆盖 DoD 全范围

任一不通过：状态保持 `已执行/验收中`，输出 `【守门未通过】T-<n>：<缺项>`。

## 核心工作流

### 阶段一 · 读取任务与会签材料

1. 读取 §当前在跑 对应 `T-n` 行：
   - 状态必须为 `已执行` 或 `验收中`
   - 读取 `acceptor_role[]` / `DoD` / `执行结果` / 历史 `验收结论`
2. 接收会签材料（见 §结论交付要素）
3. 校验会签身份：
   - 提交人必须在 `acceptor_role[]`
   - 非列表角色提交直接拒绝

### 阶段二 · 守门检查 + 会签聚合

1. 按验收守门 5 项检查本次结论
2. 不通过：
   - 保持当前状态
   - 输出缺项与补齐要求
3. 通过：
   - 追加写入 `验收结论` 列（保留会签人与日期）
   - 按多人会签语义判定状态是否扭转到 `验收中/已验收`

### 阶段三 · 冲突与阻塞升级

命中以下情况时生成 `B-x`：

- 多位验收结论互相冲突
- 存在不通过且无法归因
- 条件通过逾期未补齐
- DoD 与执行证据无法建立映射

升级路径：`→ manager`（仅裁决，不中转验收）。

## 结论交付要素（acceptor 必填）

| 要素 | 说明 |
|------|------|
| 任务编号 | `T-n` |
| 验收人 | 角色名（必须在 `acceptor_role[]`） |
| 验收时间 | `YYYY-MM-DD` |
| 结论 | `通过 / 不通过 / 条件通过` |
| DoD 对照 | 每条 DoD 的验收结果与依据 |
| 不通过定位 | DoD 条目 + 缺口 + 建议（不通过必填） |
| 条件清单 | 条件与截止时间（条件通过必填） |
| 备注 | 风险与后续建议（可选） |

## 输出格式（默认）

```text
## 【accept 输入】
## 【守门检查】（5 项）
## §当前在跑 更新（验收结论列、状态）
## 【accept 结论】
## 阻塞单（若有）
```

## 注意事项

- 本技能是**公共会签技能**，不是 manager 调度技能
- 不跨列写：仅允许写 `验收结论` 与 `状态`
- 不跨任务：单次仅处理一个 `T-n`
- 责任归属不变：调用方不能替换 `acceptor_role[]`
- 不通过不自动打回执行：先升级 manager 裁决打回方向

> 通用注意事项见 [`manager.md`](../../../manager.md) §注意事项。
